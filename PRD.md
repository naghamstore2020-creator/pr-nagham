## 1. ملخص المنتج

نظام ويب متكامل لإدارة منتجات **نغم ستور للاتصالات**، يهدف إلى ربط ومزامنة البيانات بين متجر سلة الإلكتروني والنظام المحاسبي عبر ملفات Excel. يقوم النظام بمطابقة المنتجات، تحديث المخزون، تعديل الأسعار، وتحليل الأرباح بطريقة آلية وذكية.

**الإصدار الحالي:** 2.7  
**آخر تحديث:** 2026

---

## 2. الجمهور المستهدف

- **محاسب / مدير مخزون** — يقوم بجرد المخزون وتحديث الأسعار
- **مدير النظام (ADMIN)** — صلاحية كاملة لإدارة المستخدمين والإعدادات
- **محرر (EDITOR)** — صلاحيات تشغيلية (مطابقة، جرد كامل، تسعير كامل، أرباح)
- **حساب تجريبي (DEMO)** — صلاحيات محدودة للعرض والتجربة فقط

## 4. الميزات الرئيسية

### 4.1 رفع الملفات
- رفع ملفات Excel بصيغة `.xlsx` للمتجر، النظام، والرفوف
- التحقق من صحة الملفات والبنية (الأعمدة المطلوبة، عدد الصفوف، الحجم)
- تخزين الملفات في `public/uploads/` واسترجاعها للتشغيل
- الحد الأقصى: 50MB لكل ملف

### 4.2 الجرد (Inventory)

#### 4.2.1 الجرد اليومي
- عرض انخفاض الكميات فقط (لا يسمح بزيادة الكميات)
- المنتجات بدون SKU → تصفر تلقائياً
- المنتجات غير الموجودة في ملف النظام → تصفر تلقائياً
- مخرجات: ملف Excel محدث، تقرير (تم التحديث/لم يتغير/محظور/تم التصفير)

#### 4.2.2 الجرد الكامل
- يسمح بالزيادة والنقصان في الكميات
- نفس منطق المطابقة بالـ SKU مع معالجة كاملة
- مخرجات: ملف Excel محدث، تقرير كامل

### 4.3 التسعير (Pricing)

#### 4.3.1 تحديث أسعار التكلفة
- تحديث سعر التكلفة من ملف النظام إلى ملف المتجر
- فلترة حسب الفئات (رئيسية/فرعية)
- عرض التغييرات قبل التنفيذ (القديم ← الجديد، الفرق)

#### 4.3.2 تحديث أسعار البيع
- حساب سعر البيع بناءً على: سعر التكلفة + هامش ربح + رسوم + VAT
- توحيد أسعار المتغيرات (Variants): تجميع المنتجات ذات المتغيرات حسب اللون/السعة
- آلية التوحيد: إذا لون واحد متاح → جميع المتغيرات تأخذ نفس السعر؛ إذا ألوان متعددة → أعلى سعر يفوز
- سعر الأب (Parent) = أقل سعر بين المتغيرات
- الفلترة حسب الفئات واستثناء SKU

#### 4.3.3 تسعير كامل
- تنفيذ تحديث التكلفة + حساب البيع في مرور واحد

### 4.4 مطابقة الأسماء (AI Matching)

#### 4.4.1 محرك المطابقة المحلي
محرك ذكي بالكامل يعمل محلياً (لا يحتاج API خارجي):

1. **تطبيع النص**: تحويل الأحرف العربية (أ→ا، ى→ي، ة→ه، إلخ)
2. **استخراج الكيانات**:
   - العلامة التجارية (40+ علامة: ابل، سامسونج، هواوي، شاومي، Oppo، إلخ)
   - الموديل (نظام رقمي/حرفي مثل A56، S24، 15ProMax، SP-0003)
   - سعة التخزين (128GB، 256GB، 1TB)
   - القوة الكهربائية (25W، 45W، 100W)
   - نوع المنفذ (Type-C، Lightning، Micro-USB)
   - الإصدار (Pro، Pro Max، Ultra، Plus، Lite، Mini)
   - اللون
   - نوع المنتج (جوال، كفر، سماعة، شاحن، كيبل، باوربانك، سبيكر، تابلت، ساعة)
3. **حساب التشابه**: معامل Dice + Jaccard + تطابق الكيانات
4. **التسجيل حسب الفئة**:
   - **الجوالات**: الموديل 50%، العلامة 20%، السعة 15%، اللون 5%، النص 10%
   - **الكفرات**: موديل الجهاز 40%، الإصدار 30%، العلامة 20%، النص 10%
   - **الإكسسوارات**: كود الموديل 60%، العلامة 25%، المواصفات 15%
5. **قواعد إلزامية**: تطابق الموديل ← ≥95%؛ عدم تطابق المنفذ ← حد أقصى 30%؛ عدم تطابق القوة ← حد أقصى 40%
6. **حالات المطابقة**: ≥95% تلقائي، 80-94% يحتاج مراجعة، <80% مرفوض

#### 4.4.2 التخزين المؤقت
- عند قبول مطابقة → تُحفظ في قاعدة البيانات + ملفات JSON
- عند تشغيل المطابقة مرة أخرى → المقبولات سابقاً تُحسب كـ 100% مباشرة (بدون إعادة حساب)
- المرفوضات تُحفظ كـ REJECTED ولا تُستثنى (تُعاد حسابه لإمكانية إعادة النظر)

#### 4.4.3 الواجهة
- عرض جميع المنتجات (حتى غير المتطابقة بالـ SKU)
- لكل منتج: زر قبول ورفض (مع حفظ الحالة في DB)
- إمكانية تغيير الحالة (قبول ← رفض والعكس)
- فلترة: الكل، مطابق تلقائياً، يحتاج مراجعة، مرفوض، مقبول
- فرز: نسبة التطابق (أعلى/أقل)، اسم المتجر، اسم النظام، SKU

### 4.5 تحليل الأرباح (Profit Analysis)

حساب الربح لكل منتج بعد خصم العمولة والضريبة:

1. **سعر التكلفة بعد VAT**: `costPrice × 1.15`
2. **سعر البيع بعد VAT**: `sellPrice × 1.15`
3. **العمولة**: `((سعر البيع بعد VAT × النسبة) + المبلغ الثابت) × 1.15`

| وسيلة الدفع | النسبة | المبلغ الثابت |
|---|---|---|
| **Mada** | 1% | 1 ريال |
| **Visa** | 2.2% | 1 ريال |
| **Tamara** | 6.99% | 1.5 ريال |

4. **صافي البيع**: `سعر البيع بعد VAT - العمولة`
5. **الربح**: `صافي البيع - سعر التكلفة بعد VAT`
6. تصدير: Excel و CSV

### 4.6 إدارة المستخدمين

- عرض التقدم (processedItems/totalItems)

## 5. سير العمل التشغيلي

### 5.1 سير المطابقة
```
رفع ملف المتجر + ملف النظام
        ↓
تحميل المطابقات المقبولة سابقاً من DB + ملفات الكاش
        ↓
مطابقة المنتجات الجديدة بواسطة SKU
    ├── SKU موجود في النظام → حساب نسبة التطابق
    └── SKU غير موجود ← رفض تلقائي
        ↓
عرض النتائج (قبول/رفض/تلقائي/مراجعة)
        ↓
المستخدم يقبل أو يرفض يدوياً
        ↓
حفظ الحالة في DB + ملفات الكاش
```

### 5.2 سير التسعير
```
اختيار ملف المتجر (مخزن مسبقاً)
        ↓
تحديد الفئات المطلوبة
        ↓
تحديد هامش الربح و SKU المستثناة
        ↓
معالجة التسعير:
    ├── تحديث التكلفة من ملف النظام
    ├── حساب سعر البيع
    └── توحيد أسعار المتغيرات
        ↓
معاينة النتائج قبل التنفيذ
        ↓
تنفيذ → تصدير ملف Excel محدث
```

### 5.3 سير الجرد
```
رفع ملف المتجر + ملف النظام (اختيارياً ملف الرفوف)
        ↓
اختيار نوع الجرد (يومي / كامل)
        ↓
معالجة: مطابقة بالـ SKU، حساب الفروق
        ↓
يومي: ↓ فقط / كامل: ↑ و ↓
        ↓
تصدير ملف Excel محدث + تقرير
```

## 6. هيكل قاعدة البيانات

```
User ──┬── Job ──┬── InventoryLog
       │         ├── PricingLog
       │         └── AiMatch
       └── AuditLog

ApiSetting (مستقل)
```

### الجداول:
| الجدول | المحتوى |
|---|---|
| **User** | المستخدمين (اسم/باسورد/دور/حالة) |
| **Job** | سجل الوظائف المنفذة (نوع/حالة/تقدم/أخطاء) |
| **InventoryLog** | تفاصيل تغييرات الجرد (SKU/كمية قديمة/جديدة/الفرق) |
| **PricingLog** | تفاصيل تغييرات التسعير (SKU/سعر تكلفة/بيع قديم/جديد) |
| **AiMatch** | نتائج المطابقة (اسم متجر/نظام/نسبة/حالة/SKU) |
| **AuditLog** | سجل المراجعة (إجراء/تفاصيل/مستخدم) |
